🚀 Quick Fix Guide

Fix "Auth session missing" error in 5 minutes

⚠️ Common Error

Error: "Auth session missing" or "403 Forbidden"

Cause: Database not set up OR email confirmation enabled

Fix Time: 5 minutes ⏱️

1

Open Supabase Dashboard

Go to your Supabase project:

🔗 Open Supabase Dashboard
2

Run SQL Script

This creates all required tables and adds sample data:

  1. Click SQL Editor in left sidebar
  2. Open file: QUICK-FIX-SQL.md
  3. Copy ENTIRE content (all 115 lines)
  4. Paste into SQL Editor
  5. Click ▶ RUN button
  6. Wait for "Success. No rows returned"

✅ This creates:

  • profiles table
  • diagnosis_results table
  • inventory table with 10 sample products
  • orders table
  • auto_reorders table
3

Disable Email Confirmation

Allow instant login without email verification:

  1. Click Authentication (left sidebar)
  2. Click Providers tab
  3. Find Email provider
  4. Click ⚙️ Edit (pencil icon)
  5. UNCHECK ☐ "Confirm email"
  6. Click Save

✅ Now users can login immediately!

4

Create Test User

Option A: Via Supabase (Recommended)

  1. Click Authentication → Users
  2. Click ➕ Add user (green button)
  3. Fill in:
    Email: test@medai.com Password: Test@123456 ✅ Auto Confirm User: CHECK THIS!
  4. Click Create user
  5. Go to SQL Editor and run:
    INSERT INTO profiles (id, email, full_name, role) SELECT id, 'test@medai.com', 'Test User', 'user' FROM auth.users WHERE email = 'test@medai.com' ON CONFLICT (id) DO NOTHING;

Option B: Use Signup Page

  1. Open login.html
  2. Click "Create Account"
  3. Fill form and submit
5

Test & Verify

Use the debug tool to verify everything works:

🔍 Open Debug Tool

In the debug tool:

Section 1: Click "Check Status" → All green
Section 2: Click "Test Connection" → Success
Section 2: Click "Check Tables" → 5 tables found
Section 4: Login with test@medai.com
Section 5: Click "Load Inventory" → 10 products

🎉 You're All Set!

Now you can:

👤 Login 📊 Dashboard 🛒 Shop
